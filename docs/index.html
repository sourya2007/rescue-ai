HOW IT WORKS

One signal,
four transformations.

Every transmission moves through the same fixed sequence — from raw static to a structured, actionable record dispatch can act on immediately.

STAGE 01

DSP Cleaning

Persistent white noise and static are stripped from the raw transmission using STFT-based spectral subtraction, isolating the speech signal before it ever reaches a model.

spectral subtraction · short-time fourier transform
STAGE 02

Speech-to-Text

Cleaned audio is transcribed locally — no transmission ever touches a third-party API, which matters when the content is an active incident.

OpenAI Whisper · local inference
STAGE 03

Speaker Diarization

The transcript is segmented by speaker — separating Dispatch from Unit, and Unit from Unit — so the resulting log reads like an actual exchange, not a wall of text.

Pyannote 3.1
STAGE 04

Tactical Extraction

A local LLM reads the attributed transcript and extracts an incident profile, location and hazard metadata, an urgency rating, and concrete action items for first responders.

Ollama · Llama 3.2:3b

Real-Time DSP

Processing API

  • Uvicorn
  • Unified launcher script
AI Models

Local Inference

  • Whisper (small)
  • Pyannote 3.1

SETUP & DEPLOYMENT

Running in
under five steps.

Two runtimes, one launcher. The dashboard and the AI engine start together from a single command.

  • 01
    Node.js & npm

    Runs the React dashboard.

  • 02
    Python 3.10+

    Runs the AI processing pipeline.

  • 03
    Ollama

    Install, then pull llama3.2:3b for local tactical extraction.

  • 04
    Hugging Face token

    Needed for pyannote/speaker-diarization-3.1 — accept the model terms first.

terminal — rescue-ai
$git clone https://github.com/sourya2007/rescue-ai.git && cd rescue-ai
$python -m venv .venv && source .venv/bin/activate
$pip install -r requirements.txt && npm install
$export HF_TOKEN="your_token_here"
$python backend.py
→ Dashboard http://localhost:5173 → API Server http://127.0.0.1:8000
